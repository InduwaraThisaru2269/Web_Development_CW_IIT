Home Page

  • Facts Section
    • HTML:
      • Used "facts-container" to center the facts text and image.
      • The inside adjusts dynamically to screen sizes.
    • CSS:
      • display: flex; flex-direction: column; align-items: center; ensures elements stack neatly.
  • Cards Section
    • HTML:
      • Used "cards-container"to group multiple "card" components.
      • Each card contains an image (img), a title (h3), and a “Learn More” button (a).

      • padding-bottom: 20px; ensures enough spacing below text elements.

Table

  • Table Structure
    • HTML:
      • The "table-container" wraps the table, ensuring responsiveness and a structured layout.
      • The table element holds all data, with thead for headers and tbody for the main content.
    • CSS:
      • border-collapse: collapse; removes spacing between table cells for a clean look.
      • width: 100%; ensures the table expands fully within its container.
      • overflow-x: auto; allows horizontal scrolling on smaller screens.
  • Header Section
    • HTML:
      • Uses th inside thead to define column headings.
      • First row contains the table title and spans across multiple columns.
      • Added tooltips using "tooltip" for additional information.
    • CSS:
      • background-color: #C28400; provides a distinct visual contrast for headers.
      • color: white; ensures the text is readable against the dark background.
      • font-weight: bold; enhances visibility.
  • Table Body
    • HTML:
      • Each row (tr) represents a country with relevant statistics.
      • Columns contain data for food security rate, malnutrition rate, and government support.
      • Icons (✔ & ❌) indicate positive or negative progress.
      • links direct users to detailed programs.
    • CSS:
      • nth-child(even) { background-color: #f9f9f9; } applies alternating row colors for better readability.
      • padding: 10px; ensures adequate spacing between content.
      • hover { background-color: #ffeb99; } highlights rows when hovered for interactivity.
  • Footer Section
    • HTML:
      • A single row merges all columns (colspan="4") to display a call-to-action message.
    • CSS:
      • background-color: #FFD700; makes the footer message stand out.
      • text-align: center; centers the message visually.
      • font-weight: bold; ensures emphasis.

Challenges and Lessons Learned

  • Layout Issues

    Aligning elements and creating complex layouts can be tricky, especially when working with different CSS properties such as float, flexbox, or grid. Debugging layout issues, especially with nested elements or complex designs, was time-consuming and required a solid understanding of CSS layout techniques.

    • Learned how to effectively use Flexbox and Grid for better alignment.
    • Discovered the importance of browser developer tools for debugging.
  • Text and Font Rendering

    Different devices and browsers render fonts in slightly different ways, leading to text being displayed differently across various screens. Ensuring legibility and aesthetic consistency, especially with custom web fonts, was a challenge.

    • Gained an understanding of font fallback strategies.
    • Used CSS properties like `font-smoothing` and `letter-spacing` for better readability.
  • Maintaining Consistency

    As students implementing CSS for the first time, maintaining consistent design elements (such as colors, typography, and spacing) was challenging.

    • Created a style guide to maintain uniformity.
    • Used CSS variables for easier theme management.
