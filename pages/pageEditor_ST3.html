    Challenges and Lessons Learned

    • Implementing Hover Effects

      Ensuring smooth and responsive hover effects presented multiple challenges. Some elements did not transition properly, particularly the hidden member details that needed to appear on hover. The main issue stemmed from improper display properties and conflicts with z-index. Additionally, achieving smooth transitions required careful adjustments to CSS properties like opacity, transform, and transition-duration. After multiple refinements, hover effects were optimized to create a seamless and visually engaging user experience.

      • Learned how to effectively use hover transitions and Grid for better alignment.
      • Discovered the importance of browser developer tools for debugging.
    • Text and Font Rendering

      Different devices and browsers render fonts in slightly different ways, leading to text being displayed differently across various screens. Ensuring legibility and aesthetic consistency, especially with custom web fonts, was a challenge.

      • Gained an understanding of font fallback strategies.
      • Used CSS properties like `font-smoothing` and `letter-spacing` for better readability.

      Link to the Page: Team Page

    • Maintaining Layout Consistency

      Keeping the layout structured and responsive across different screen sizes was another challenge. The initial grid-based structure caused elements to break or overlap on certain screen resolutions. Implementing grid-template-columns helped maintain balance, but additional adjustments using flexbox, padding, and media queries were necessary to ensure proper alignment and spacing. Extensive testing across various screen sizes helped refine the layout for better adaptability and visual appeal.

      • Created a style guide to maintain uniformity.
      • Used CSS variables for easier theme management.
    • Maintaining Layout Consistency

      Choosing an effective color scheme required careful consideration to maintain both aesthetics and accessibility. Some initial color choices resulted in poor contrast, making text difficult to read, while others disrupted the overall design harmony. Finding the right balance involved adjusting the color palette to improve readability, using high-contrast color pairings, and ensuring adequate spacing. Adhering to Web Content Accessibility Guidelines (WCAG) standards was also a priority to ensure inclusivity for all users.
